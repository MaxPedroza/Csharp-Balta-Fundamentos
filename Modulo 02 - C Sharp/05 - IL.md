# IL (Intermediate Language)

---

## O que é IL?

**IL (Intermediate Language)** é o código intermediário gerado pelo compilador C# (e outras linguagens .NET) antes de virar código de máquina.

Ele é:

- Independente de plataforma (não é específico de Windows, Linux, etc.).
- Mais baixo nível que C#, porém mais alto nível que o código de máquina.
- Entendido pelo **CLR**, que depois o converte (via JIT) para instruções nativas.

---

## Por que usar IL?

O uso de IL traz vários benefícios:

- Permite que **várias linguagens** (C#, VB.NET, F#, etc.) rodem sobre a mesma plataforma.
- Facilita otimizações em tempo de execução.
- Ajuda em cenários de inspeção e análise de código (ferramentas de engenharia reversa, por exemplo).

---

## Exemplo simples

Quando você escreve algo como:

```csharp
int x = 10;
int y = 20;
int z = x + y;
```

O compilador C# gera IL equivalente a essas operações, e só depois o CLR/JIT converte esse IL para instruções específicas da CPU.

Você pode, inclusive, usar ferramentas (como ILDASM ou ILSpy) para enxergar o IL gerado a partir do seu código.

---

## Resumo

- IL é a **linguagem intermediária** do .NET.
- Serve como ponte entre o código C# e o código de máquina.
- É um dos pilares que permite que o .NET seja **multilíngue** e **multiplataforma**.
