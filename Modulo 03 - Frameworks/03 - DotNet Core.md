# .NET Core

---

## Por que o .NET Core foi criado?

Com o crescimento da **nuvem** e de servidores **Linux**, a Microsoft precisava de um framework que:

- Rodasse em **Windows, Linux e macOS**.
- Fosse mais **leve**, **modular** e **performático**.
- Evoluísse mais rapidamente que o .NET Framework tradicional.

Assim nasceu o **.NET Core**.

---

## Características principais

- **Multiplataforma**: Windows, Linux, macOS.
- **Open source**: código aberto, com contribuições da comunidade.
- **Instalação lado a lado**: várias versões do .NET Core podem coexistir na mesma máquina.
- Foco em **performance** e **segurança**.

Versões importantes (até o momento descrito no material original):

- 1.x – primeiras versões, com conjunto básico de bibliotecas.
- 2.x – amadurecimento da plataforma.
- 3.1 – versão considerada bem completa e marcada como **LTS**.

---

## Tipos de aplicações com .NET Core

- APIs e aplicações web (ASP.NET Core).
- Aplicações de console.
- Serviços em nuvem (Azure, AWS, etc.).
- Em versões mais recentes, também desktop e outras categorias.

---

## Recomendação da Microsoft

Para **novas aplicações**, a recomendação é usar .NET Core (ou já a versão unificada .NET 5+), e **não** o .NET Framework clássico.

Isso garante:

- Melhor suporte futuro.
- Acesso às últimas melhorias de linguagem e runtime.
- Possibilidade de hospedar em múltiplos sistemas operacionais.
