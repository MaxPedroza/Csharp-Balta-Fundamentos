# Frameworks

---

## Por que falar de frameworks?

Começar uma aplicação do zero é trabalhoso: é preciso lidar com arquivos, banco de dados, segurança, memória, logs e muito mais.

Em vez de reinventar a roda, usamos **frameworks** que já trazem:

- Código testado e reutilizável.
- Padrões e boas práticas embutidos.
- Integração com diversas tecnologias.

No mundo Microsoft, o principal framework para desenvolvimento é o **.NET**.

---

## O que é um framework?

Um **framework** é:

- Uma **estrutura** sobre a qual construímos nossas aplicações.
- Um **conjunto de bibliotecas e ferramentas** prontas para uso.
- Um **alicerce** que resolve problemas comuns para que você foque na regra de negócio.

Exemplos de coisas que um framework .NET oferece:

- Leitura e escrita de arquivos.
- Acesso a banco de dados.
- Comunicação via rede (HTTP, TCP, etc.).
- Gerenciamento de memória.

---

## Vantagens de usar frameworks

- **Produtividade**: você escreve menos código "infrainstrutura".
- **Segurança**: reaproveita componentes amplamente testados.
- **Padronização**: todo o time segue a mesma base tecnológica.
- **Evolução**: atualizações do framework trazem melhorias de performance e recursos.

Na prática, quase sempre começamos um projeto **sobre um framework**, e não "do zero".

---

## Frameworks no ecossistema .NET

Ao longo deste módulo você vai ver:

- **.NET Framework** (clássico, focado em Windows).
- **.NET Core** (multiplataforma, mais moderno).
- **.NET Standard** (padrão de APIs em comum).
- **.NET 5+** (unificação da plataforma .NET).

Esses nomes representam diferentes fases da evolução do mesmo ecossistema.
